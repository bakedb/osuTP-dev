Tournament System Guide

Welcome to your osu! tournament portal. This system uses a bracket-based structure with qualifiers to determine player placement and progression through different difficulty levels.

Explore beatmaps

Tournament Phases

Progressive tournament structure:

  • Phase 1: Qualifiers - Determine initial placement
  • Phase 2: Bracket Play - Head-to-head competition
  • Phase 3: Finals - Championship matches

Qualification Process

Map Selection: Choose one map from each bracket

Progression: Pass maps to advance to next bracket

Assignment: Placed based on highest completed bracket

Difficulty Brackets

Normal: Entry-level for beginners

Hard: Intermediate difficulty

Insane: Advanced players

Expert: High-level competition

Expert+: Very high difficulty

Expert++: Maximum challenge

Match Format

Best of 7: First to 4 map wins

Map Pool: 7 maps per match

Tiebreaker: Final map if 3-3

Scoring: Accuracy (80%) + combo (20%) based

Tournament Features

Live Updates

Real-time score updates and match progress with automatic refresh every 30 seconds.

Interactive tournament tree showing player progression and match results.

Search, filter, and sort all tournament beatmaps by difficulty and length.

Getting Started

For Participants

  • Sign in to access participant features
  • Review qualification requirements
  • Check match schedules and brackets
  • Practice with tournament beatmaps
  • Ensure technical setup is ready

For Organizers

  • Use main.py to manage beatmaps
  • Update tournament-data.json for live scores
  • Monitor bracket assignments
  • Track player progression
  • Maintain match schedules

Scoring System

Base Formula: ((0.8 accuracy) 100) + ((0.2 (combo/max_combo)) 100)

Mod Multipliers: EZ (0.50x), HR (1.12x), HD (1.05x), FL (1.10x)

Dual Display: Original score + scaled score (1'000'000 max)

Use the score calculator in main.py to estimate match performance.

Technical Information

File Structure

  • index.html: Main tournament website
  • beatmaps.json: Beatmap database
  • tournament-data.json: Live tournament data
  • main.py: Management tools

Data Flow

  • main.py updates JSON files
  • JSON files store all data
  • Website reads and displays data
  • Auto-refresh keeps everything current

Tournament rules

  • Eligibility: Registered participants only.
  • Fair play: No cheating, no unauthorized mods, and no unapproved clients.
  • Respect: Harassment or toxicity results in disqualification.
  • Timing: Be on time; late arrivals may forfeit matches.
  • Technical: Verify your input, audio, and overlays before matches.
  • Skins: Inquire about approved skins before the tournament starts. You can get all approved skins here.
  • Mods: No Fail should be enabled during all matches. Other mods will be determined before the round begins.

Checklist

  1. Check ruleset (if first round) Check if the skin is valid, if the maps are unmodified and updated, and the correct client is being used.
  2. Check mods Enable No Fail and any other mods specified on the beatmap list.
  3. Verify scores Use the scoring algorithm below to determine the score of the play and compare against the other player.

Scoring

((0.8 * [Acc to 4SF]) * 100) + ((0.2 * sqrt([combo] / [max combo])) * 100)

You can find a Python utility for scores and more related to osu!TP on the GitHub. Download and run main.py. The Python file requires no additional dependencies, only a relatively new Python interpreter that supports Tkinter.
